Dino AI 新老 App 用户迁移方案

明确老 App 用户是否迁移至新 App，系统对比「迁移」与「不迁移」两条路径的优势 & 劣势，并给出可落地的渐进式迁移策略与成功指标。

版本 v1.0 更新 2026-07-09 适用：韩国 / 沙特 / 马来（+ 印尼 / 越南）
00

结论速览

先看核心判断，再看下方逐项论证。

✅
综合结论：本次迁移属于「低风险、高收益」场景，推荐执行迁移，并采用「渐进式灰度」而非强制一刀切。 新老 App 在登录方式、会员权益、国家覆盖上高度兼容，迁移的核心障碍（账号匹配 / 权益缩水 / 地区缺失）都很低。

🟢 为什么可以迁

  • 登录方式完全一致手机号 / kakao / 谷歌邮箱 / Facebook / AppID，账号体系可无缝对齐
  • 会员权益 1:1 可映射新 App 仅新增「周包」，老用户月包 / 年包权益不缩水
  • 国家全覆盖老 App 三国均被新 App 覆盖，不存在迁过去无法服务的地区

🔴 为什么值得迁

  • 双端并行成本高开发 / 客服 / 运营长期双份投入，边际效率低
  • 数据割裂用户与付费数据分散两端，难形成统一增长视图
  • 存量无法激活老用户无法导入印尼 / 越南等新市场增长体系
01

背景与基础信息

新老 App 在登录、会员、国家三个维度的对比，是本次迁移判断的事实基础。

维度老 App新 App差异点
登录注册方式 手机号、kakao、谷歌邮箱、Facebook、AppID 手机号、kakao、谷歌邮箱、Facebook、AppID 完全一致 ✅
会员类型 月包、年包 周包、月包、年包 新增「周包」
覆盖国家 韩国、沙特、马来 韩国、沙特、马来、印尼、越南 新增「印尼、越南」

关键解读：登录方式一致 → 账号匹配风险极低；会员仅做增量（周包）→ 老用户权益可平滑映射且可作二次运营抓手；国家仅做增量扩张（印尼、越南）→ 迁移属于「向上兼容」，不牺牲任何存量能力。
02

迁移方案（推荐路径）

把老 App 用户引导 / 迁移至新 App，逐步下线老 App。以下为优势与劣势（含缓解措施）。

迁移的优势

  • 保留存量用户老用户直接转化为新 App 用户，省去重新拉新的高成本
  • 保留付费用户月包 / 年包订阅关系延续，降低付费流失
  • 体验统一用户集中在新 App，享受周包等新功能与后续迭代
  • 单端维护降本逐步下线老 App，减少双端开发 / 客服 / 运营成本
  • 数据归一行为 / 付费数据统一沉淀，利于分析与增长决策
  • 承接新市场复用新 App 已覆盖印尼 / 越南的统一投放与增长体系
  • 周包引流低门槛周包可对老用户做转化 / 复购二次运营

迁移的劣势 / 风险

  • 账号匹配失败缓解：以登录方式为唯一键映射 + 人工兜底核对
  • 订阅态迁移复杂缓解：苹果 / Google / 第三方支付按平台规则迁移，提前对齐
  • 进度 / 数据丢失缓解：迁移前全量备份 + 抽样对账（≥98% 一致率）
  • 迁移感知打断缓解：一键迁移引导页，减少操作步骤
  • 老用户流失缓解：迁移激励，如赠送周包 / 续费折扣
  • 客服咨询上升缓解：提前备 FAQ、客服 SOP、迁移专属入口
  • 跨国数据合规缓解：韩国 / 沙特 / 马来隐私合规分区域核对
  • 重复购买风险缓解：订阅去重校验（目标 < 0.5%）
03

不迁移方案（备选路径）

老 App 与新 App 并行运营，不强制用户迁移。以下为优势与劣势。

不迁移的优势

  • 零迁移风险无账号匹配 / 订阅迁移 / 数据丢失等技术风险
  • 用户无打扰老用户维持原体验，不产生迁移流失
  • 节奏可控新 App 专注印尼 / 越南新市场快速验证
  • 短期投入低无需迁移工具 / 对账 / 客服 SOP 等一次性成本

不迁移的劣势

  • 双端长期维护开发 / 测试 / 客服 / 运营长期双份成本
  • 数据割裂用户 / 付费数据分散，难形成统一增长视图
  • 体验分裂周包等新功能无法惠及老用户，体验逐渐落后
  • 存量无法激活老 App 存量用户无法导入新市场增长体系
  • 品牌认知混乱同品牌双 App 易造成用户与市场认知混乱
  • 技术债累积老 App 维护成本随时间上升，最终仍需迁移，越晚越贵

05

结论与建议

推荐执行迁移

采用「渐进式灰度迁移」，而非强制一刀切

核心理由：

  1. 登录方式一致 + 会员权益 1:1 映射 + 国家全覆盖 → 迁移的核心风险都很低。
  2. 双端并行的长期成本，远高于一次性迁移成本。
  3. 新 App 已具备「承接老用户 + 拓展新市场」的能力，迁移能让数据与增长归一。

分阶段推进，先小范围验证再全量，降低风险、可随时回退。

阶段 3 · 扩大期

评估收尾

前置阻塞：订阅迁移口径需在灰度期前与苹果 / Google / 第三方支付渠道拉齐；账号匹配规则（以登录方式为唯一键）需与账号侧确认，避免归因断链。
